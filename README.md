# Workpad (상담 워크패드)

고객센터 상담사를 위한 **완전 로컬, 오프라인** 플로팅 카드 메모 도구. 상담사들이 메모장·스티커 메모 여러 개를 펼쳐놓고 쓰는 워크플로우(참조정보 보기 → 콜 메모 → 상담 시스템 붙여넣기)를, 보안 환경에서 그대로 쓸 수 있게 다시 설계했습니다.

> 포지셔닝: **메모장의 자유도는 그대로, 관리 능력만 더한다.**

## 왜

- 듀얼 모니터 한쪽을 메모장 창 여러 개가 덮고 있는 게 전국 공통 풍경
- 클라우드 노트(원노트·노션)는 보안 정책으로 불가 + 창을 "펼쳐놓고" 못 씀
- 지금은 고객 개인정보가 평문 `.txt`로 좌석 PC에 방치 — 보안 사각지대

## 핵심 기능 (v0.1)

- **독립 카드 창**: 각 메모가 별도 창. 상담 시스템 모달·브라우저 옆에 자유롭게 겹쳐 배치. 항상 위 토글, 접기, 자유 크기/위치
- **줄 = 버튼**: 상용구 카드에서 줄을 한 번 클릭하면 복사(아이콘은 호버 시에만, 복사 후 체크). 더블클릭하면 그 줄 편집, 드래그하면 평소처럼 텍스트 선택 — 셋이 충돌하지 않음
- **빈 줄 더블클릭 = 수집**: 카드 맨 아래 빈 줄을 더블클릭하면 클립보드 내용이 플레인 텍스트로 그 자리에 붙고 새 빈 줄이 생김
- **복사 스탬프**: 복사할 때 `[06/12 14:35] 내용`처럼 날짜·시간·상담사ID를 자동으로 붙임(카드별 서식 설정, Shift+클릭은 원문만)
- **자동 저장·복원**: 저장 버튼 없음. 입력 즉시 저장, 크래시·재부팅 후 카드 내용+위치+크기 그대로 복원
- **컨트롤 패널**: 카드 생성, 전체 표시/숨김, 배치 프리셋 저장·복원
- **콜 메모 자동 파기**: 지정한 일수가 지난 콜 메모 줄을 자동 삭제

## 보안 설계

- **외부 통신 제로** — 네트워크 코드 없음, CSP로 원격 리소스 차단
- **로컬 암호화 저장** — AES-256-GCM, 데이터 키는 Windows DPAPI(Electron `safeStorage`)로 보호
- **자동 파기** — 콜 메모는 TTL 경과 시 자동 삭제(스냅샷 포함)
- 오픈소스(MIT) — 코드 공개로 보안 검토 가능, 텔레메트리 없음

## 개발 실행

```bash
npm install
npm start
```

Windows 10/11 + Node.js 18+ 필요. (배포 빌드/포터블 패키징은 로드맵 참조)
