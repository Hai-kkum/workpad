# 상담 워크패드 — 설계 문서 (DESIGN)

문서 버전 0.1 · 2026-06-14 · 이 문서가 요구사항·설계의 단일 기준(source of truth)입니다.
구현 현황 기준선: Electron **v0.1**(동작 중, 유지). 이 문서는 v0.1에서 누락된 요구사항을 드러내고 다음 단계를 정의합니다.

---

## 1. 목적 · 포지셔닝

상담사들이 메모장·스티커 메모로 때우는 데스크탑 워크플로우(**참조정보 보기 → 콜 메모 → 상담시스템 붙여넣기**)를, 보안 폐쇄망에서 쓸 수 있는 **완전 로컬 오픈소스 "플로팅 카드" 도구**로 대체한다.

> 포지셔닝: **메모장의 자유도는 그대로, 관리 능력만 더한다.**
> 공개 카테고리: "더 좋은 스티커 메모"가 아니라 **보안 환경용 상담사 데스크탑 워크벤치**.

## 2. 사용자 · 사용 맥락

- 고객센터 상담사. 대부분 **듀얼 모니터**, 한쪽 절반이 메모장 창 3~4개로 덮여 있음.
- 좌석 PC는 개인 소유 아님 + 보안 정책 → **클라우드 노트(원노트·노션) 사용 불가**, 인터넷 차단 가능, 설치 권한 제한 가능.
- 상담 시스템(레거시 그리드/모달)·브라우저를 **동시에** 봐야 함. 메모는 그 창들 사이/위에 끼워 배치.
- 현재 리스크: 고객 개인정보가 평문 `.txt`로 좌석에 방치, 파기 정책 없음 → **보안 사각지대**.

## 3. 절대 원칙 (메모장의 강점을 깨면 도구의 존재 이유가 사라진다)

1. **즉시성** — 로딩·로그인·저장 버튼 없음. 키 입력이 곧 저장.
2. **조각조각(독립 창)** — 단일 대시보드 금지. 각 메모가 독립 창으로, 다른 앱 위/옆에 자유 배치.
3. **드래그 자유** — 본문은 항상 네이티브 텍스트 선택 가능. 원클릭 복사는 그 위에 얹는 가속기일 뿐, 그 아래 드래그 선택은 100% 살아 있어야 한다.

### 4.9.1 마스킹 모델 전환 + 맥락 워터마크 (2026-06-15 결정 — 다음 세션 구현)

배경: 파괴적 마스킹은 상담사가 가져온 PII(카드번호 등)를 결제·확인에 **못 쓰게 만들어 업무를 막음**(콜메모에 카드 가져왔는데 마스킹돼 결제 불가). → 비파괴 + 필요 시 공개로 전환.

- **SE-6 비파괴 공개형 마스킹:**
  - 원본은 **암호화 저장**, 화면엔 마스킹 표시(기본 가림 = 어깨너머·화면공유 보호)
  - **줄/셀 클릭 복사 → 원본 그대로** (결제 시스템에 붙여넣기 가능; 눈으로 안 봐도 처리됨)
  - **👁 토글 → 해당 값 일시 공개**, 수 초 후 **자동 재마스킹**(노출 최소화)
  - **TTL 자동 파기**로 복원 가능 PII 단명화
  - 드래그 선택 복사는 화면(마스킹)값이 잡힘 — 원본은 **클릭 복사 경로로만**(설계상 일관)
  - **카드번호 PCI 정책 결정 필요:** (a) 짧은 TTL 강제 + 공개 이력 기록, 또는 (b) 저장 안 함(세션 중만 보유 후 파기) — 보안팀과 확정
- **SE-7 맥락 워터마크 (공개 시에만):**
  - 평소 **워터마크 0**(상담사 불편 없음). **👁로 PII 공개하는 순간** 그 카드에 옅은 워터마크(상담사ID·좌석/PC명·시각) → 재마스킹 시 사라짐

### 4.9.2 SE-9 선택적 패스프레이즈 잠금 (2026-06-19 결정 — B-9 재결정)

기존 결정(B-9 "패스프레이즈 KDF skip, 무로그인 즉시성 우선")을 **재결정**: 잠금을 **선택(opt-in, 기본 off)** 으로 두면 기본 즉시성은 그대로라 절대원칙 #1과 무관(사용자 지적). → 구현 확정.

- **기본 off** = 현행(DPAPI 자동, 비번 없음, 즉시성). **on** = 사용자가 설정에서 패스프레이즈 지정.
- **보호 강도 = 강함(사용자 확정)**: 잠금 on이면 데이터 키(DEK)를 **scrypt(패스프레이즈)+AES-256-GCM**으로 래핑(DPAPI 대신). 같은 PC·계정이라도 비번 없이 복호 불가(진짜 암호화). 인프라는 백업(SE-8, `main.js` scrypt N=16384) 재사용.
- **잠금 시점 = 실행 시만(사용자 확정)**: 앱 시작 시 1회 잠금 해제 프롬프트 → 성공해야 카드 로드. 실행 중 자동/수동 재잠금은 범위 밖(추후). 패닉 숨김(Ctrl+Alt+H)은 별개로 유지.
- **복구 없음**: 패스프레이즈 분실 = 데이터 복구 불가(백도어 없음). 설정 시 명시 경고 + 백업 권장.
- 활성/변경/해제: 설정에서. 켤 때 기존 DEK를 패스프레이즈로 재래핑(데이터 보존), 끌 때 현재 비번 확인 후 DPAPI 래핑으로 복귀.
- **상태**: 설계 확정(2026-06-19), 구현 대기.

- **스택 최종 결정 = 열린 항목.** Tauri vs Electron 비교·벤치마크는 [RESEARCH.md](RESEARCH.md) §2~3 참고. 요약: 제품 정체성(보안 검토 가능·경량)엔 Tauri가, 잠긴 좌석 배포 예측성·즉시성엔 Electron이 유리. **렌더러(카드 UI) 코드는 어느 쪽이든 대부분 이식 가능** → 본 설계의 요구사항은 스택 중립. 결정 전까지 v0.1(Electron)을 기준선으로 기능 갭을 메운다.
- **Alt-Tab 숨김**: Electron=owner-window(소유 창) 트릭 / Tauri=WS_EX_TOOLWINDOW Win32. (스택별 구현 상이, 요구사항 동일)

## 7. 데이터 모델 (현행)

암호화 JSON 단일 상태: `{ settings, cards{ id→card }, presets }`.
- card: `{ id, type, title, bounds{x,y,w,h}, alwaysOnTop, collapsed, visible, format{enabled,template}, createdAt, updatedAt, (content|lines), ttlDays? }`. lines = `{text, t?}` (콜메모는 t=수집 시각).
- TTL 파기 정책: 콜메모 줄 단위, 기준=줄 수집 시각 t(없으면 카드 createdAt 폴백 = fail-closed). 줄 수정 시 t 미갱신(원 수집 시각 유지 — 의도, GAP B-4). 시각기준 선택(통화시작)·표 카드 rows는 v0.2.
- 저장 위치: `%APPDATA%\Workpad\workpad-data.enc`, 키: `workpad-key.bin`(safeStorage 래핑).
- 표 카드(예정): `rows: [[cell,…],…]`, 복사 단위=셀/행/범위 → TSV.
